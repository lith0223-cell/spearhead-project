[프로젝트 개요]

목적: 중급자 이상의 헬스인을 위한 점진적 과부하 맞춤형 웹앱(PWA) 개발

핵심 가치: 이전 운동 볼륨 기록의 직관적 확인, 1RM 자동 계산, 식단 매크로(탄/단/지) 직접 입력 및 비율 확인

기술 스택 (권장): React(또는 Next.js), Tailwind CSS, 로컬 스토리지(초기 MVP용) 또는 Firebase(무료 DB)

[주요 화면 및 기능 명세]

1. 대시보드 (메인 화면)

오늘의 운동 루틴 바로가기 버튼.

오늘 하루치 총 섭취 칼로리 및 탄/단/지 섭취 비율 그래프(통계) 표시.

2. 루틴 관리 및 운동 수행 화면 (1~2단계 핵심)

루틴 생성: 최대 7개까지 요일별/부위별 루틴 세팅 및 보관 가능.

이전 기록 연동: 특정 종목(예: 벤치프레스) 선택 시, UI 하단이나 옆에 '직전 운동 수행 기록(무게/횟수/세트)'이 희미하게 혹은 작게 표시되어야 함.

세트 진행: >     * 무게 단위(kg/lb) 토글 버튼.

수행한 무게와 횟수를 입력하면 해당 종목의 1RM을 공식에 따라 자동 계산하여 표시.

세트 추가 및 삭제 기능.

타이머 기능 (Screen Wake Lock API 필수 적용):

세트 '완료 체크' 클릭 시, 설정해둔 휴식 시간(기본 30초 단위 조정 가능) 타이머 자동 시작.

타이머 종료 시 브라우저 내장 효과음(Beep 음) 발생.

해당 종목의 모든 세트 완료 시, 다음 운동 종목 리스트로 자동 스크롤 또는 화면 전환.

3. 식단 기록 화면 (3단계)

입력 폼: [끼니명(아침/점심/저녁/간식)] + [메뉴명 텍스트] + [탄수화물 g] + [단백질 g] + [지방 g] 입력칸 제공.

자동 계산 로직: 탄, 단, 지 g수를 입력하면 해당 끼니의 칼로리를 (탄*4)+(단*4)+(지*9)로 자동 계산하여 실시간으로 띄워줌.

통계 누적: 입력된 끼니별 데이터가 하루치 총량으로 합산되어 대시보드에 반영됨.
