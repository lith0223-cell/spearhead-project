# 제품 요구사항 문서 (PRD) - 프로젝트 헬스 (가제)

본 문서는 중급자 이상의 헬스인을 위한 점진적 과부하 맞춤형 웹앱(PWA)인 '프로젝트 헬스'의 핵심 기능, 화면 구성, 기술 스택 및 개발 제약사항을 정의합니다. 별도의 서버 비용 없이 프론트엔드(Next.js)와 구글 시트 API만을 활용하여 안전하고 확장성 있는 MVP를 구축하는 것을 목표로 합니다.

## 2. 사용자 시나리오 (User Scenario)

* **누가 (Who)**
  * 운동 기록의 체계화, 점진적 과부하 달성, 그리고 탄/단/지 식단 관리에 진심인 **중급자 이상의 헬스 마니아**.
  * 단순히 운동 횟수를 채우는 것이 아니라, 지난 수행 기록을 바탕으로 철저하게 무게와 볼륨을 계산하여 증량해 나가는 스마트한 유저.
* **언제 (When)**
  * **웨이트 트레이닝 중**: 헬스장에서 무거운 바벨을 내려놓고 숨을 고르는 세트 사이의 휴식 시간.
  * **식사 및 영양 섭취 시**: 아침, 점심, 저녁 혹은 운동 전후로 닭가슴살, 고구마 등 식단을 챙겨 먹고 영양 성분을 기록하는 순간.
* **왜 (Why)**
  * 매 세트마다 지난주 기록을 찾기 위해 메모장을 뒤적거리거나 기억을 더듬는 번거로움을 없애기 위해.
  * 손에 땀이 찬 상태에서 폰 화면이 자꾸 꺼져 비밀번호를 다시 입력해야 하는 스트레스를 방지하기 위해.
  * 세트 완료 후 매번 타이머를 따로 켜야 하는 불편함을 자동화하기 위해.
  * 하루 동안 목표한 탄/단/지 매크로 비율을 직관적인 시각적 그래프로 즉시 확인하고 맞춤형 영양 공급을 하기 위해.

## 3. 핵심 기능 목록 (Feature List)

### 3.1 필수 기능 (MVP - 운동 기록 중심)
* **대시보드 (메인 화면)**
  * [오늘의 운동 시작하기] 거대하고 직관적인 루틴 바로가기 버튼 제공.
* **루틴 관리 시스템**
  * 요일별 또는 부위별 맞춤형 루틴 세팅 및 보관 기능 (**최대 7개**로 제한).
* **운동 수행 및 기록 (캐러셀)**
  * **이전 기록 자동 연동**: 특정 종목 선택 시 직전 운동의 성공 기록(무게/횟수/세트)을 UI에 흐릿하게 자동 노출.
  * **단위 토글**: 무게 단위를 kg과 lb 간에 실시간 자유 전환 가능.
  * **1RM 실시간 자동 계산**: 무게와 횟수 입력 시 에플리(Epley) 공식에 의거하여 예상 1RM을 실시간 산출 및 표시.
  * **세트 유연성**: 운동 중 손쉽게 세트를 추가하거나 삭제할 수 있는 기능.
* **운동 편의 기능 (자동화)**
  * **자동 화면 전환**: 한 종목의 모든 세트 완료 체크 시, 별도 조작 없이 다음 운동 종목으로 자동 슬라이드 전환.
  * **휴식 타이머 카운트다운**: 세트 완료 체크 즉시 사전에 설정된 휴식 시간이 화면에 큼직하게 카운트다운되며 시작 (30초 단위 조절 가능).
  * **오디오 알림**: 타이머가 `00:00`에 도달하면 브라우저 내장 효과음(Beep음)을 1초간 재생하여 휴식 종료 알림.
  * **화면 꺼짐 방지**: 운동 수행 뷰 전용 Screen Wake Lock API를 필수 적용하여 디바이스 화면 상시 유지.

### 3.2 선택 기능 (후순위 확장 - 식단 및 DB 확장)
* **식단 기록 및 영양소 관리**
  * 끼니별 입력 폼: [끼니명(아침/점심/저녁/간식)] + [메뉴명] + [탄수화물(g)] + [단백질(g)] + [지방(g)] 입력 칸 제공.
  * **칼로리 자동 계산 로직**: 탄/단/지 입력 시 `(탄수화물 * 4) + (단백질 * 4) + (지방 * 9)` 공식으로 실시간 칼로리 산출.
  * **대시보드 통계 연동**: 하루 총 섭취 칼로리와 탄/단/지 비율을 메인 대시보드 그래프에 실시간 누적 반영.
* **데이터 영속성 확장**
  * 초기에는 브라우저 로컬 스토리지에 데이터를 임시 캐싱 및 저장하고, 최종 제약사항에 맞춰 구글 시트 API와 실시간 동기화.

## 4. 기술 스택 (Tech Stack)

* **프론트엔드 프레임워크**: `Next.js` (App Router 또는 Pages Router)
  * SSR/SSG를 통한 빠른 초기 로딩 속도 확보 및 API Routes를 활용한 보안 서버리스 백엔드 구축 가능.
* **스타일링 라이브러리**: `Tailwind CSS`
  * 유틸리티 퍼스트 기반의 신속한 UI 개발 및 모바일 퍼스트 반응형 레이아웃 구성.
* **데이터베이스 및 저장소**: `Google Sheets API` & `Local Storage`
  * 유료 DB 서버 없이 오직 구글 시트만을 영속성 저장소로 활용. 클라이언트 단에서는 속도 향상을 위해 로컬 스토리지를 캐시로 병행 사용.
* **브라우저 내장 Web API**:
  * `Screen Wake Lock API`: 운동 중 디바이스 화면 잠금 및 꺼짐 현상 강제 차단.
  * `Web Audio API` (또는 `HTML5 Audio`): 외부 미디어 파일 없이 코드로 순수 Beep 음 효과음 구현.

## 5. 화면 구성 (Wireframe & UI Structure)

본 앱은 **모바일 퍼스트 반응형 디자인** 구조를 가지며, 화면 하단의 고정 탭 바(Bottom Navigation Bar)를 통해 이동합니다.

### 5.1 대시보드 화면 (메인 탭)
* **상단**: 오늘 날짜, 현재 요일 및 맞춤형 동기부여 인사말 노출.
* **중앙**: 오늘의 탄/단/지 섭취 비율 원형/바 그래프 및 총칼로리 요약 (식단 기능 구현 전에는 '준비 중' 프리뷰 혹은 더미 데이터 표시).
* **하단**: **[오늘의 운동 시작하기]** 형태의 가독성 높은 거대 볼드 버튼 배치.

### 5.2 루틴 관리 및 운동 수행 화면 (운동 탭)
* **루틴 목록 뷰 (기본 상태)**
  * 요일별/부위별 카드 형태의 루틴 리스트 (최대 7개 격자/세로 배열).
  * 각 카드 내 [편집], [삭제] 기능 및 최하단 [새 루틴 추가] 버튼 배치.
* **운동 수행 뷰 (루틴 시작 시 화면 전환) 🌟**
  * **상단 영역**: 현재 수행 중인 운동 종목명 및 상단 슬라이드 인디케이터(예: ○ ● ○ ○ ○) 표시.
  * **중앙 캐러셀 영역**: 좌우 스와이프 제스처 및 좌우 화살표 버튼을 통해 **이전/다음 운동으로 부드러운 슬라이드 이동** 지원.
    * *직전 기록 가이드*: 입력 폼 하단에 저대비 회색 글씨로 '지난번: 60kg x 10회' 노출.
    * *기록 입력 폼*: [세트 번호] | [무게 입력창] | [횟수 입력창] | [완료 체크박스].
    * *실시간 정보*: 현재 세트에 입력된 수치 기준 예상 1RM 실시간 계산 노출.
  * **하단 고정 영역**:
    * 무게 단위 [kg / lb] 토글 스위치.
    * **휴식 타이머 바**: 세트 완료 체크박스 활성화 시 즉시 카운트다운 숫자가 큼직하게 표시되며 타이머 바가 시각적으로 줄어듦. (기본 30초부터 시작하여 30 / 60 / 90 / 120초 순으로 배수 변환 가능).
    * 스와이프가 익숙하지 않은 사용자를 위한 보조용 `[이전 운동]`, `[다음 운동]` 하단 버튼 배치.

### 5.3 식단 기록 화면 (식단 탭 - 후순위 확장)
* **상단**: 오늘 총 섭취 칼로리 목표 대비 현재 누적 칼로리 요약 바.
* **중앙**: 아침/점심/저녁/간식 타임라인 리스트 및 세부 메뉴, 탄/단/지 성분 표시.
* **하단**: 식단 추가 플로팅 버튼(FAB) 제공, 클릭 시 입력 팝업 모달 오픈.
